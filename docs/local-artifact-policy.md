# Local artifact policy

This repository can produce large boot-image and USB-staging artifacts during real hardware deployment. Those files are intentionally local-only and should not be committed.

## Ignored generated outputs

The following paths are generated by boot-image or USB deployment work and are ignored by git:

- `boot-image/aports/` — downloaded Alpine kernel/initrd and package/cache material
- `boot-image/rootfs/` — expanded root filesystem used while building the boot image
- `boot-image/*.cpio.gz` — generated initramfs archives
- `boot-image/*.img` — generated bootable disk images such as `hermes-boot.img`
- `boot-image/*.iso` — locally downloaded installer ISO material
- `boot-image/*.tar.gz` — local packaged boot-image archives
- `usb-backup-*/` — local snapshots copied from removable media
- `/auto-deploy.sh` — accidental root-level copy of `boot-image/overlay/auto-deploy.sh`

## Files that should not be hidden by ignore rules

Hand-written diagnostics or recovery scripts should be reviewed intentionally instead of hidden by a broad ignore rule. Examples:

- `boot-image/ROOT-CAUSE-ANALYSIS.md`
- `boot-image/fix-uefi-boot*.sh`
- changes to tracked scripts such as `boot-image/make-boot-image.sh`

If these are useful, move them into `docs/` or a reviewed script path and open a focused PR. If they are only local scratch work, archive them outside the repository or delete them after backup.

## Safe cleanup flow

Before cleaning a working tree, preserve evidence and avoid deleting by accident:

```bash
git status --short
git diff -- boot-image/make-boot-image.sh > /tmp/hermes-bootstrap-make-boot-image.patch
mkdir -p ~/archives/hermes-bootstrap-local-artifacts
# Move only local generated/scratch files after review; do not move tracked files unless you intend to reset them.
```

Then decide separately whether to:

1. keep a focused patch for review,
2. archive scratch files outside the repo, or
3. remove generated artifacts that can be recreated.
